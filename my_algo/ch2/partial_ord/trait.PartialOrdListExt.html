List trait的一个扩展trait, 提供了一些基于偏序的方法.

Provided methods

fn search(&self, x: &Item) -> Option<usize>

二分搜索. 在有序表中搜索元素x, 返回最大的不超过x的元素序号, 若所有元素都比x大, 则返回None.

Correnctness

此方法要求表有序(且为顺序).

fn merge_mid<'a>(&'a self, rhs: &'a Self) -> Option<&'a Item>

给出两个有序表的中位数(即两个有序表合并为新的有序表后的中位数).

Correctness

此方法要求两个表有序且为顺序.

fn mid(&self) -> Option<&Item>

给出有序表的中位数(第⌈len / 2⌉个).

Correctness

此方法要求表有序(且为顺序).

fn mid_between(&self, a: usize, b: usize) -> Option<&Item>

在[a, b)范围内计算中位数(第⌈(a + b) / 2⌉个).

Correctness

此方法要求表有序(且为顺序).

fn merge(self, rhs: Self) -> Self where
    Self: Sized + Default, 

合并两个有序表, 得到一个新的有序表.

Correctness

此方法要求表有序(且为顺序).

算法

从头部开始删除元素(先行逆置以避免不必要的移位), 将较小者插入新的表, 直到两个表都为空. 该算法的时间复杂度为O(n_1 + n_2), 空间复杂度为O(n_1 + n_2).

fn dedup_sorted(&mut self)

对有序表去重. 这是一个保序的算法.

Correctness

此方法要求表有序(且为顺序).

fn delete_between_unsorted(&mut self, x: &Item, y: &Item) -> Vec<Item>

删除表中值介于x, y之间(含)的所有元素(x < y), 返回被删除的元素列表. 这是一个保序的算法, 但返回的列表并不保持顺序. 注意: 该方法与delete_between不同, 不要求表是有序的.

fn delete_between(&mut self, x: &Item, y: &Item) -> Vec<Item>

删除有序表中值介于x,y之间的所有元素(x < y, 不含x及y), 返回被删除的元素列表. 这是一个保序的算法, 但返回的列表并不保持顺序.

Correctness

此方法要求表有序(且为顺序).

fn delete_between_opt(&mut self, x: &Item, y: &Item, contain: bool) -> Vec<Item>

删除有序表中值介于x,y之间的所有元素(x < y, 若contain为true则包含x及y), 返回被删除的元素列表. 这是一个保序的算法, 但返回的列表并不保持顺序.

Correctness

此方法要求表有序(且为顺序).

算法的正确性依赖于交换循环的不变式: j始终指向首个需要被删除的元素, k始终指向首个不需要被删除的元素.

快速排序中的helper function.

Panics

fn sort_between(&mut self, begin: usize, end: usize)

对begin~end之间的元素进行快速排序.

快速排序.

寻找第一个极小元的位置. 若表空, 则返回None.

fn delete_min(&mut self) -> Option<Item>

删除第一个极小元. 若表空, 则返回None. 这不是一个保序的算法.
